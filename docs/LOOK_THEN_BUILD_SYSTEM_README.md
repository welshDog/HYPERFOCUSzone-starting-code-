# 🔥 BROski Ultra LOOK-THEN-BUILD System 🚦

**BROski Level: LEGENDARY | Status: MANDATORY FOR ALL AGENTS**
_Last updated: 2025-07-08_

---

## 🧠 PURPOSE

**Prevent wasted effort, duplicate features, and chaotic builds!
Every AI agent, coder, or contributor MUST follow this ultra-simple, ADHD-friendly workflow before starting ANY new build.**

*The "Memory Crystal system" is a shared knowledge base that tracks all features, modules, and changes across the project. For more details, see [Memory Crystal System Documentation](./memory_crystals/README.md) or consult the relevant onboarding materials.*

---

## ⚡ THE LEGENDARY LOOK-THEN-BUILD WORKFLOW

1. **SCAN & MAP PHASE**
    - Agent **scans the full project folder** and the Memory Crystal system before starting any new feature or task.
    - Index all features, files, and partially-built modules.
    - Check for anything similar to the requested build in H:\

2. **REPORT & RECOMMEND PHASE**
    - Agent **generates a summary report**:
        - List all relevant features found (with status: complete, partial, broken).
        - Highlight “almost the same” features.
        - **Recommend:** Build new, upgrade, merge, or skip!

3. **APPROVE & PROCEED PHASE**
    - Agent **asks for approval** from BROski X, ARIA, or human (you):
        - “Feature already exists — should I build new, upgrade, or skip?”
    - **NO building happens without this step!**

4. **BUILD & EXECUTE PHASE**
    - Only after approval, agent builds, upgrades, or merges as needed.
    - Avoids double work, chaos, and overwriting good stuff.

    - For Boardroom system changes, manually edit and update `BOARDROOM_SYSTEM_README.md` to reflect the latest changes, and use the provided script or tool to update the latest Boardroom Memory Crystal JSON file (e.g., `memory_crystals/BOARDROOM_MEMORY_CRYSTAL_YYYYMMDD.json`), ensuring both are fully synchronized.
    - After any change, **update the Memory Crystal system** so all agents know what’s new.
    - For Boardroom system changes, update both `BOARDROOM_SYSTEM_README.md` and the latest Boardroom Memory Crystal (e.g., `memory_crystals/BOARDROOM_MEMORY_CRYSTAL_YYYYMMDD.json`).
